# Online Retail Store Database System

## Overview
This project models the backend database system of an online retail store.
It focuses on **relational database design, query optimization, and analytical SQL** commonly used in real-world systems.

## Tech Stack
- Database: PostgreSQL / MySQL
- Language: SQL

## Key Features
- Normalized relational schema (3NF)
- Order and inventory management
- Analytical queries for sales and customer insights
- Stored procedures and triggers for automation
- Indexing and performance optimization

## Database Design
Entities included:
- Customers
- Products
- Orders
- Order Items
- Inventory
- Suppliers

Relationships are enforced using foreign keys.

## SQL Concepts Used
- JOINs
- Common Table Expressions (CTEs)
- Window Functions
- Stored Procedures
- Triggers
- Indexes
- Transactions

## Sample Analyses
- Top-selling products
- Monthly sales trends
- High-value customers
- Supplier performance analysis

## Use Case
This project simulates how backend services interact with a database to:
- Process orders
- Maintain inventory consistency
- Generate business insights

## How to Run
1. Execute `schema/create_tables.sql`
2. Insert data using `data/insert_sample_data.sql`
3. Run queries from the `queries/` folder

This project is intended to demonstrate **database engineering and analytical SQL skills**.
